# Ocean · 产品设计文档

> 版本：v0.3 · 阶段：Demo 迭代中
> 面向读者：产品设计者 + 开发者

---

## 一、产品概述

**软件名称**：Ocean

**一句话定位**：专为女同情侣设计的 LGBT 友好双人目标规划软件，以航海为核心视觉隐喻，帮助两个人把共同的心愿变成可以每天推进的计划。

**核心价值主张**：
- 把长期目标拆解成每天可感知的行动，让规划有方向感
- 双人共创、共同打卡，让每一步进展都有「我们一起」的仪式感
- 以航海可视化呈现成就，让两人的努力变成一片不断扩展的海域

**目标用户**：女同情侣，中文用户为主

**目标平台**：移动端优先（iOS / Android），技术栈待定

---

## 二、产品背景与动机

现有情侣类 app 大多以记录恋爱纪念日、聊天为主，缺乏对「共同目标」的深度支持。对于 LGBT 情侣而言，还额外面临出行、异地、法律等现实挑战，需要更多结构化的规划工具。

Ocean 希望填补这个空白：不只是记录感情，而是帮两个人真正把想法落地。

**用户痛点：**
- 有很多共同想法，但总是停在「想法」阶段，没有推进
- 各自忙碌，缺乏一个共同的地方对齐进度
- 现有任务管理工具（如 Todoist、Notion）是为个人设计的，缺少双人协作的情感温度
- LGBT 情侣的特定场景（出国公证结婚、移居 LGBT 友好城市等）在通用工具里得不到支持

## 三、核心功能模块

### 3.1 账号配对系统

两人各自注册账号，通过邀请码完成配对，形成一个「航行小队」。配对后两人共享同一片海域、同一艘船，所有数据实时同步。

**流程：** 注册 → 创建账号 → 生成邀请码 → 伴侣输入邀请码 → 配对成功 → 进入主页

### 3.2 心愿池

心愿池是两人许愿的地方，存放还没有发展成完整计划的想法和心愿。

**核心逻辑：**
- 两人都可以往心愿池投入新想法
- 投入时 AI 会自动给出初步建议，用户可编辑
- 心愿支持**持续编辑**——用户可以随时打开一个心愿，修改标题、调整步骤、补充备注，像草稿一样慢慢完善，不需要一次性填完
- 当想法成熟时，任意一方都可以将其「转化为计划」，转化后从心愿池移走，进入计划清单
- 心愿池强调「收集感」和「孵化感」，允许想法在池中慢慢生长

### 3.3 计划清单

计划清单存放所有正在推进的计划，每个计划都有明确的阶段和截止日期。

**三层任务结构：**

```
计划（例：去成都旅行）
  └── 阶段（例：攒钱 · 截止3月）
        └── 每日打卡记录（文字 + 图片）+ 共享进度条
  └── 阶段（例：定机酒 · 截止4月）
  └── 阶段（例：做旅游攻略 · 截止5月）
```

**阶段管理：**
- 创建计划时，AI 自动将目标拆解为若干阶段
- 用户可以手动编辑阶段名称、调整顺序
- 每个阶段可设定截止日期（DDL）
- 共享进度条由两人共同推进，可拖动调整完成度

**计划类型：**
- 未来规划（如：出行、移居、公证结婚）
- 共同成长（如：一起运动、学习新技能）

### 3.4 AI 步骤拆解

当用户创建计划或整理心愿池中的想法时，AI 根据具体内容自动生成个性化的阶段拆解建议。

**特别支持 LGBT 场景**，例如：
- 「去泰国公证结婚」→ 拆解为：了解当地法律要求、准备材料、预订公证机构、安排行程
- 「移居 LGBT 友好城市」→ 拆解为：调研城市、评估工作机会、规划搬家

用户可以在 AI 建议的基础上自由增删、编辑阶段内容。

> **当前状态**：Demo 中为写死模板，正式版需接入真实 Claude API

### 3.5 双人打卡

在计划清单中，每个当期阶段支持两人每天各自记录打卡内容，共同推进进度条。

**打卡界面结构：**
- 顶部：阶段名称 + 共享进度条 + 截止日期
- 中间：聊天对话框式打卡记录流，两人记录交织排列，按时间顺序显示
- 底部：输入框（支持文字 + 图片）+ 发送

**进度条：**
- 两人共享一根进度条
- 可手动拖动调整完成百分比
- 进度更新对双方实时可见

### 3.6 航海可视化系统

航海画面是 Ocean 的情感核心，代表两人共同建造的世界。

**视觉规则：**
- 一艘船在海上，两个可定制小人站在船上
- 完成阶段 → 船向前航行，解锁新的海域
- 海域随成就不断扩展，已解锁的区域可以探索和回顾
- 完成重要里程碑时触发特殊庆祝画面和动画

**成就与里程碑：**
- 里程碑与航海进度绑定，无金币系统
- 完成一定数量的阶段可解锁新的**图鉴卡片**（卡片式收集，未解锁的卡片灰显，解锁后亮起并有动画）
- 图鉴内容示例：LGBT 友好国家/城市图标卡、旅行目的地卡、特殊成就卡等
- 里程碑动态生成，反映用户实际完成的内容

### 3.7 角色定制系统

两人各自定制自己在船上的小人形象，形象贯穿整个 app 的互动场景。

**中度定制内容：**
- 肤色
- 发型
- 发色
- 服装款式
- 配饰
- 表情

**双人代表色：** 两人各自选择专属颜色，贯穿进度条、打卡气泡、小纸条等全局 UI 元素。

### 3.8 小纸条（双人互动）

小纸条是两人之间的私下互动通道，绑定在具体任务下。

**规则：**
- 在某个任务内，任意一方可以给对方写一张小纸条
- 小纸条私下发送，只有对方可见
- 对方下次打开 app 时看到提示
- 内容可以是鼓励、提醒、吐槽，语气轻松温柔

## 四、交互流程

### 流程一：新用户配对
```
注册账号 → 生成邀请码 → 分享给伴侣 → 伴侣输入邀请码 → 配对成功 → 定制角色形象 → 进入主页
```

### 流程二：投入心愿池
```
主页 → 心愿池 Tab → 点击添加 → 输入想法 → AI 自动生成建议步骤 → 用户编辑确认 → 存入心愿池
```

### 流程三：心愿转化为计划
```
心愿池 → 选择一个想法 → 点击「转化为计划」→ 完善阶段内容和 DDL → 确认发布 → 想法从心愿池移走，进入计划清单
```

### 流程四：日常打卡
```
首页（航海画面）→ 下拉展开 → 今日打卡列表（当期阶段）→ 进入某个阶段 → 聊天框写记录 + 拖动进度条更新完成度
```

**今日打卡逻辑：**
- 有当期阶段（当月有截止日期的阶段）→ 显示当期阶段列表 + 每日打卡入口
- 无当期阶段 → 展示所有进行中计划的大目标概览

### 流程五：小纸条
```
打开某个任务 → 点击「写纸条」→ 输入内容 → 发送 → 对方下次打开 app 时看到提示 → 私下查看
```

### 流程六：里程碑 & 航海进度
```
完成若干阶段 → 触发里程碑 → 船前进 / 解锁新海域 → 庆祝动画 → 可能解锁新船外观或角色配饰
```

## 五、UI 设计规范

### 5.1 整体视觉方向

**参考风格**：Spiritfarer（手绘水彩插画，有光感和流动感，黄昏光晕，色彩有情绪和深度）

**关键词**：唯美浪漫、黄昏光感、有情绪、温暖但不轻浮、有使用欲望

**UI 语言**：全局波浪形设计语言——卡片边缘带轻微波浪弧度，分区用波浪曲线而非直线切割，按钮和标签都有有机弧度，避免方方正正的直线感

**任务清单结构参考**：Todoist（层次清晰、功能密度适中）

**互动卡片风格参考**：Finch（圆润卡片、柔和背景、插画点缀）

### 5.2 色彩系统

**主色系**：黄昏海面——深蓝紫的海 + 暖珊瑚点缀，有情绪、有深度

| 色彩角色 | 描述 | 示例用途 |
|---------|------|---------|
| 深蓝紫 | 带紫调的深海蓝，有神秘感 | 海面主色、深色背景、导航栏 |
| 暖珊瑚 | 柔和橙粉，有温度 | 按钮、强调色 |
| 雾粉 | 天边云霞的粉紫色 | 天空渐变层、辅助色 |

**双人代表色**：用户自选，贯穿进度条、聊天气泡、小纸条等全局元素。

### 5.3 字体

- **中文正文**：系统默认中文字体（苹方 / Noto Sans SC）
- **标题**：衬线字体，有书卷气（Noto Serif SC 或同类）
- **英文 / 数字**：DM Sans 或同类几何无衬线

### 5.4 组件风格

- 卡片圆角：大圆角（16px 以上），温暖有质感
- 阴影：轻微投影，不硬朗
- 插画元素：船、海浪、云、海鸥等手绘元素点缀各页面
- 按钮：圆角大、有轻微纹理或渐变，避免纯色扁平

### 5.5 首页结构

```
第一屏：航海画面（全屏手绘水彩场景，两个小人站在船上）
下拉后：今日打卡列表 / 计划概览（Todoist 结构 + Finch 卡片风格）
```

### 5.6 导航栏（底部 Tab）

| Tab | 图标 | 内容 |
|-----|------|------|
| 首页 | 船锚 | 航海画面 + 下拉今日打卡 |
| 心愿池 | 许愿瓶 | 浏览、添加、转化想法 |
| 计划清单 | 地图 | 所有进行中的计划和阶段 |
| 我们 | 双人剪影 | 小纸条、里程碑记录、账号设置 |

## 六、数据结构定义

### User（用户）
```
id            // 唯一标识
name          // 昵称
avatarConfig  // 角色定制配置（肤色、发型、发色、服装、配饰、表情）
color         // 双人代表色（用户自选）
partnerId     // 配对伴侣的 user id
```

### Pair（配对关系）
```
id            // 唯一标识
userAId       // 用户 A
userBId       // 用户 B
createdAt     // 配对时间
ship          // 船的当前状态（外观、解锁的海域等）
```

### Wish（心愿）
```
id            // 唯一标识
pairId        // 所属配对
createdBy     // 创建者 user id
title         // 心愿标题
aiSuggestion  // AI 给出的初步建议
status        // "in_pool" | "converted"
createdAt
```

### Plan（计划）
```
id            // 唯一标识
pairId        // 所属配对
title         // 计划名称
type          // "future" | "grow"
sourceWishId  // 来源心愿 id（如果是从心愿池转化而来）
stages        // 阶段列表
createdAt
```

### Stage（阶段）
```
id            // 唯一标识
planId        // 所属计划
title         // 阶段名称
deadline      // 截止日期
progress      // 共享进度（0-100）
order         // 排序
```

### CheckIn（打卡记录）
```
id            // 唯一标识
stageId       // 所属阶段
userId        // 打卡用户
content       // 文字内容
images        // 图片列表
createdAt
```

### Note（小纸条）
```
id            // 唯一标识
stageId       // 绑定的阶段
fromUserId    // 发送者
toUserId      // 接收者
content       // 内容
isRead        // 是否已读
createdAt
```

## 七、AI 功能规格

### 7.1 当前状态（Demo）

步骤拆解为写死的模板，根据类型（心愿 / 计划 / 成长）返回固定结构，无个性化。

### 7.2 目标状态（正式版）

接入 Claude API，根据用户输入的具体内容，生成真正个性化的阶段拆解建议。

**调用时机：**
- 用户在心愿池添加新想法时
- 用户将心愿转化为计划时

**Prompt 设计要点：**
- 明确说明这是女同情侣的规划工具，AI 应理解 LGBT 特定场景
- 输出格式为结构化的阶段列表（阶段名 + 建议截止时长）
- 语气温暖、具体，避免通用模板感
- 建议步骤数量控制在 3-6 个，不宜过多

**示例输入/输出：**

输入：「去泰国清迈公证结婚」

输出建议阶段：
1. 了解清迈公证结婚的法律要求和所需材料（建议：1个月内）
2. 准备护照、户籍证明等材料（建议：1个月内）
3. 联系并预订当地公证机构（建议：2个月内）
4. 预订机票和住宿（建议：2个月内）
5. 出发完成公证手续（建议：按出行日期）

## 八、待解决问题清单

| # | 问题 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | AI 步骤拆解是假的 | 🔴 必须 | 接入真实 Claude API，支持 LGBT 特定场景 |
| 2 | 没有账号系统，身份靠手动切换 | 🔴 必须 | 需要完整的注册、登录、配对流程 |
| 3 | 数据不持久化，刷新即消失 | 🔴 必须 | 需要后端存储和数据同步 |
| 4 | 里程碑内容写死 | 🔴 必须 | 改为动态成就系统，与航海进度绑定 |
| 5 | 心愿池缺少转化流程 | 🔴 必须 | 心愿 → 计划的转化入口和流程完全缺失 |
| 6 | 没有航海可视化 | 🔴 必须 | 当前只是普通任务列表，缺少核心情感体验 |
| 7 | 没有通知推送系统 | 🟡 之后 | 缺少打卡提醒，可能导致用户流失 |
| 8 | 金币系统 | ✅ 已决定取消 | 成就感由航海可视化承载，金币系统移除 |

## 九、后续迭代方向（V2+）

- **通知推送系统**：DDL 临近提醒、伴侣打卡通知、小纸条到达通知
- **成就博物馆**：回顾两人走过的所有完成计划，像一本航海日志
- **航海地图扩展**：随使用时长解锁更多海域、岛屿、天气变化等视觉彩蛋
- **LGBT 友好资源库**：在相关计划类型下，内嵌实用信息（如 LGBT 友好国家指南、公证结婚攻略）
- **国际化**：在中文版成熟后，考虑面向海外华人及东南亚 LGBT 社群扩展

## 十、版本迭代记录

### v0.3 · 2026年4月（Bug修复 + 视觉优化）

**Bug 修复：**
- 首页打卡卡片点击进入后无法返回，已修复返回逻辑
- 心愿池「继续完善」页面无法添加或删除步骤，已补充添加步骤和删除步骤功能

**视觉优化：**
- 整体色调从「过暗的黄昏」调整为「明亮黄昏」——保留橙金暖感但大幅提亮，避免沉闷感
- 成就图鉴卡片视觉重新设计，提升美观度和收集感

---

### v0.2 · 2026年4月（设计迭代）

**UI 视觉方向调整：**
- 整体色调从「晴天海洋」改为「黄昏海面」——深蓝紫海 + 橙金天光 + 暖珊瑚，有情绪、有深度
- 确立全局**波浪形设计语言**：所有卡片边缘带轻微波浪弧度，分区用波浪曲线切割，避免方正直线感
- 主视觉参考 Spiritfarer 的光晕感和层次感，追求真实光感而非扁平插画

**功能逻辑调整：**
- 心愿池支持**持续编辑**：心愿可以像草稿一样随时打开修改，慢慢完善，不需要一次性转化
- 里程碑改为**卡片式图鉴**：解锁一张亮一张，内容包含 LGBT 友好国家/城市图标等，未解锁灰显

**Demo 体验调整：**
- 减少内置示例数据，只保留最少量（1个心愿 + 1个计划），避免呆板的测试数据感
